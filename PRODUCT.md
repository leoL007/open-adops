# OpenAdOps 产品定义

## 一句话定位

OpenAdOps 是面向海外广告优化师的本地优先 AI 投放决策工作台：把客户零散信息和投放数据转化为可执行、可追溯、可交付的广告决策。

## 目标用户

- 同时负责金融、游戏、工具等多个行业的海外广告优化师。
- 需要在 Google Ads、Meta Ads、TikTok Ads 之间制定分工的代理商团队。
- 经常收到不完整 Offer、简短客户策略、媒体报表和 MMP 报表的项目负责人。

## 核心任务

1. 接收客户 Offer 和零散约束。
2. 区分已确认事实、合理推断和缺失信息。
3. 形成投放策略、Campaign 架构、素材生产任务和监测方案。
4. 在上线前暴露阻塞项并留下检查证据。
5. 把素材与投放假设转化为有样本门槛、停止条件和结果记录的实验。
6. 复用已确认的报表字段映射，并按两个独立日期区间计算同口径变化。
7. 上线后依据媒体与 MMP 数据诊断问题、记录动作和验证结果。
8. 将一次项目经验沉淀为可复用的行业与媒体 Playbook。

## 核心链路

```text
Offer / 客户策略 / 会议记录
  -> Brief 与客户追问
  -> 策略初稿
  -> 投放执行方案
  -> 素材生产与广告搭建
  -> Experiment Ledger 实验与学习
  -> 数据诊断与调优
  -> 报告与 Playbook
```

## 产品原则

- **不伪造确定性**：预算、KPI、日期、合规结论缺失时必须显式标记。
- **先可判断，再自动化**：每个建议都要有输入证据、负责人和验证标准。
- **跨行业统一结构**：行业差异进入 Playbook，不为每个行业重写产品。
- **跨媒体适配**：通用 Brief + Google / Meta / TikTok 媒体适配层。
- **本地优先**：浏览器保存项目；API Key 不进入前端；真实客户资料默认不上云。
- **人工批准**：不直接修改真实广告账户，不绕过项目负责人和客户确认。
- **版本可追溯**：每个正式版本都有版本号、Changelog、Tag、Release 和验收记录。
- **统计计算由代码完成**：样本量、周期和相对变化由确定性代码计算，AI 只负责实验假设与执行语境。
- **原始明细短暂存在**：CSV 原始行只在当前页面解析，项目仅持久化聚合指标、区间结果和字段可用性。

## 当前不做

- 自动调整真实广告账户预算或出价。
- 未经批准自动发布 Campaign 或素材。
- 将通用行业基准伪装成客户目标。
- 默认把原始客户资料上传到第三方云存储。
- 用功能数量替代真实工作流价值。

## 版本成功标准

一个功能只有同时满足以下条件才能进入正式版本：

1. 对应至少一个真实高频工作卡点。
2. 有固定输入案例和可验证输出。
3. Mock 与 Codex 输出使用同一份 JSON Schema。
4. 自动化测试通过。
5. 不破坏已有本地项目数据。
6. README、Changelog 和版本号同步更新。
