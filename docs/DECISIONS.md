# 产品决策记录

## ADR-001：本地优先，而不是先做云端 SaaS

- 状态：已采纳。
- 原因：客户资料和媒体导出可能敏感；当前首先验证工作流价值。
- 结果：浏览器保存项目，本地 Node Bridge 调用 Codex，静态 Demo 使用 Mock。

## ADR-002：暂不直接写广告账户

- 状态：已采纳。
- 原因：自动预算、出价和 Campaign 写入具有真实业务与合规风险。
- 结果：OpenAdOps 输出建议、搭建蓝图和检查证据，最终执行由优化师批准。

## ADR-003：缺失、推断、确认必须分开

- 状态：已采纳。
- 原因：客户 Offer 经常不完整，AI 容易把合理猜测写成事实。
- 结果：Brief 字段包含 confirmed / inferred / missing 状态和来源证据。

## ADR-004：行业差异使用 Playbook，媒体差异使用 Adapter

- 状态：已采纳。
- 原因：金融、游戏、工具与 Google、Meta、TikTok 的组合会持续增加。
- 结果：核心 Schema 保持统一，行业与媒体规则独立扩展。

## ADR-005：版本发布必须经过 PR 和验收

- 状态：已采纳。
- 原因：自动迭代需要速度，也需要稳定性与回溯能力。
- 结果：开发分支 -> 自动测试 -> Draft PR -> 合并 main -> Tag -> GitHub Release -> Pages 验证。

## ADR-006：v0.3 优先 Launch Pack，而不是继续增加输入表单

- 状态：已采纳。
- 原因：v0.2 已经能整理输入；下一步价值在于生成可交付、可搭建、可检查的投前产出。
- 结果：新增 Campaign 蓝图、预算计划、素材生产 Brief、监测方案、上线阻塞项和首周计划。
