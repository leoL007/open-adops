# 产品决策记录

## ADR-001：本地优先，而不是先做云端 SaaS

- 状态：已采纳。
- 原因：客户资料和媒体导出可能敏感；当前首先验证工作流价值。
- 结果：浏览器保存项目，本地 Node Bridge 调用 Codex，静态 Demo 使用 Mock。

## ADR-002：暂不直接写广告账户

- 状态：已采纳。
- 原因：自动预算、出价和 Campaign 写入具有真实业务与合规风险。
- 结果：OpenAdOps 输出建议、搭建蓝图和检查证据，最终执行由优化师批准。

## ADR-003：缺失、推断、确认必须分开

- 状态：已采纳。
- 原因：客户 Offer 经常不完整，AI 容易把合理猜测写成事实。
- 结果：Brief 字段包含 confirmed / inferred / missing 状态和来源证据。

## ADR-004：行业差异使用 Playbook，媒体差异使用 Adapter

- 状态：已采纳。
- 原因：金融、游戏、工具与 Google、Meta、TikTok 的组合会持续增加。
- 结果：核心 Schema 保持统一，行业与媒体规则独立扩展。

## ADR-005：版本发布必须经过 PR 和验收

- 状态：已采纳。
- 原因：自动迭代需要速度，也需要稳定性与回溯能力。
- 结果：开发分支 -> 自动测试 -> Draft PR -> 合并 main -> Tag -> GitHub Release -> Pages 验证。

## ADR-006：v0.3 优先 Launch Pack，而不是继续增加输入表单

- 状态：已采纳。
- 原因：v0.2 已经能整理输入；下一步价值在于生成可交付、可搭建、可检查的投前产出。
- 结果：新增 Campaign 蓝图、预算计划、素材生产 Brief、监测方案、上线阻塞项和首周计划。

## ADR-007：v0.4 优先 Experiment Ledger，而不是先做格式兼容

- 状态：已采纳。
- 原因：Launch Pack 已经能定义素材与测试变量，但上线后缺少假设、样本门槛、结果和学习的统一记录；XLSX、英文 UI 等能力不能独立形成广告决策闭环。
- 结果：新增跨媒体实验队列、确定性样本计算、原生实验设置提示、结果记录和报告回流；工作流可移植性调整到后续版本。

## ADR-008：OpenAdOps 不自行宣布统计显著性

- 状态：已采纳。
- 原因：Google、Meta、TikTok 的流量分配、竞价和原生实验方法不同，简单读取聚合报表不能替代平台实验报告。
- 结果：代码只计算比例指标的计划样本和周期；真实胜负仍依据媒体原生实验结果、业务护栏和项目负责人批准。

## ADR-009：素材模块优先解决生产交接，而不是继续展示通用建议

- 状态：已采纳。
- 原因：素材角度和 Hook 只有进入数量、规格、负责人、截止日期与交付状态，才能成为团队可执行的任务。
- 结果：素材页升级为生产任务清单；AI 负责生成方向，优化师负责确认交付字段，代码负责迁移、持久化和导出。

## ADR-010：先复用数据口径，再扩展文件格式

- 状态：已采纳。
- 原因：同一媒体和 AppsFlyer 报表会反复导入，重复校对字段与手工计算周期变化是高频成本；直接支持更多文件格式不能解决口径复用问题。
- 结果：v0.5.0 先交付工作区级映射模板与确定性日期对比；XLSX 继续通过导出 CSV 进入工作台，原始行不持久化。
